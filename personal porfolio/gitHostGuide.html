What is GitHub Pages?

GitHub Pages is a free static site hosting service provided by GitHub that allows you to host websites directly from your GitHub repositories. It's an excellent solution for developers who want to showcase their projects, create portfolio websites, or host documentation without any cost.

Why Choose GitHub Pages?

  • Completely Free: No hosting fees, ever
  • Easy Integration: Works seamlessly with your GitHub workflow
  • Custom Domains: Use your own domain name
  • HTTPS Support: Automatic SSL certificates
  • Version Control: Track all changes to your website
  • Reliable: Backed by GitHub's infrastructure

Step 2: Create a New GitHub Repository

  1. Log into your GitHub account
  2. Click the "+" icon in the top-right corner
  3. Select "New repository"
  4. Choose a repository name (e.g., my-awesome-website)
  5. Make sure it's set to "Public"
  6. Check "Add a README file" (optional)
  7. Click "Create repository"

üí° Pro Tip: If you name your repository username.github.io (replacing "username" with your GitHub username), it will be accessible at https://username.github.io directly!

Step 3: Upload Your Files

You can upload your files using two methods:

Method A: Manual Upload (Beginner-Friendly)

  1. In your repository, click "uploading an existing file"
  2. Drag and drop your files or click "choose your files"
  3. Select all your website files and folders
  4. Add a commit message like "Initial website upload"
  5. Click "Commit changes"

Method B: Git CLI (Recommended for developers)

Terminal Commands 
# Clone your repository
git clone https://github.com/username/repository-name.git

# Navigate to the directory
cd repository-name

# Copy your website files here
# Then add all files
git add .

# Commit your changes
git commit -m "Add website files"

# Push to GitHub
git push origin main

Step 4: Enable GitHub Pages

  1. Go to your repository on GitHub
  2. Click on the "Settings" tab
  3. Scroll down to the "Pages" section in the left sidebar
  4. Under "Source," select "Deploy from a branch"
  5. Choose "main" branch and "/ (root)" folder
  6. Click "Save"

Tips for Avoiding Common Errors

üö´ 404 Error - Page Not Found

Cause: Missing index.html file

Solution: Ensure your main HTML file is named exactly index.html (lowercase)

üé® CSS/JS Not Loading

Cause: Incorrect file paths

Solution: Use relative paths like ./style.css instead of absolute paths

üì± Images Not Displaying

Cause: Case-sensitive file names or wrong paths

Solution: Check file extensions and ensure paths match exactly

‚è±Ô∏è Changes Not Visible

Cause: GitHub Pages caching

Solution: Wait 5-10 minutes or try hard refresh (Ctrl+F5)

Quick Debugging Checklist

  • Check that your repository is public
  • Verify GitHub Pages is enabled in settings
  • Ensure index.html exists in the root directory
  • Check browser console for error messages
  • Try accessing your site in an incognito window
